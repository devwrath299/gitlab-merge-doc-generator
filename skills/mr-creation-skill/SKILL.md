# Merge Request Creation Skill

You are responsible for **creating GitLab merge requests** as the final step of the automated documentation pipeline.

---

## MR CREATION SPECIFICATIONS

### API Endpoint

```
POST /api/v4/projects/:id/merge_requests
```

### Required Payload

```json
{
  "source_branch": "docs/auto-update-<mr_iid>-<timestamp>",
  "target_branch": "development",
  "title": "docs: auto-update documentation for MR !<original_mr_iid>",
  "description": "<auto-generated description>",
  "labels": "documentation,automated",
  "remove_source_branch": true
}
```

### MR Description Template

```markdown
## 📝 Automated Documentation Update

This merge request was automatically generated by the Documentation Automation Pipeline.

### Trigger
- **Source MR:** !<original_mr_iid> — "<original_mr_title>"
- **Author:** <original_mr_author>
- **Merged at:** <merge_timestamp>

### Documentation Changes
<list of doc files created/updated/deleted>

### Change Summary
<brief LLM-generated summary of what documentation was updated and why>

---

> ⚙️ *This MR was created automatically. Please review the documentation changes before merging.*
```

---

## RULES

1. **Target branch is always `development`** — never target `main` or `master` directly.
2. **Set `remove_source_branch: true`** — clean up feature branches after merge.
3. **Add labels** `documentation` and `automated` for easy filtering.
4. **Include the original MR reference** in both title and description for traceability.
5. **Handle duplicates** — if an MR from the same source branch already exists (409 Conflict), update it instead of failing.

---

## ERROR HANDLING

| Scenario | Action |
|----------|--------|
| 201 Created | Return MR URL — success |
| 409 Conflict (branch exists) | Find existing MR for that branch, update it |
| 401 Unauthorized | Log token error, abort |
| 404 Not Found | Log project ID error, abort |
| 422 Unprocessable | Log validation error (likely branch doesn't exist remotely) |

---

## RESPONSE

Return a structured result:

```json
{
  "success": true,
  "mr_url": "https://scm.intermesh.net/<project_path>/-/merge_requests/<new_iid>",
  "mr_iid": <new_iid>,
  "source_branch": "docs/auto-update-<mr_iid>-<timestamp>",
  "files_changed": ["docs/05_api_documentation/...", "docs/04_configuration/..."]
}
```
